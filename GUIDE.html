Drame Brothers Home & Yard

Website & email setup — maintenance guide

How It Works

The website is a set of static HTML files hosted for free on GitHub Pages at dramebrothers.com. There is no database, no subscription platform, and no monthly fee. When a client submits a booking request or fills out the feedback form, the data is sent to a small Google Apps Script program which emails you the details.

Booking request flow

  1. Client fills out the estimate form at dramebrothers.com/home-cleaning.html
  2. Price updates live as they fill in room counts and options
  3. Client checks "Ready to book" — contact and scheduling fields appear
  4. Client clicks "Request Booking" — data is sent to the Apps Script
  5. Apps Script sends you a booking alert email with all their details

Feedback flow

  1. You send the client the link: dramebrothers.com/feedback.html
  2. Client fills out the form and submits
  3. Apps Script sends you a feedback summary email

Making Common Changes

Change the base rate (price per room point)

Open config.js. Find:

baseRate: 16,

Change 16 to your new rate. Save and upload to GitHub.

Change an add-on price

Open config.js. Find the addOns array and change the price value for the relevant add-on:

addOns: [ { id: 'add_fridge', label: 'Inside refrigerator cleaning', price: 25 }, { id: 'add_oven', label: 'Inside oven cleaning', price: 25 }, // ← change here { id: 'add_cabinets', label: 'Inside cabinet cleaning', price: 40 }, { id: 'add_dishes', label: 'Dish washing', price: 20 } ]

Save and upload to GitHub. The form and rates page update automatically.

Add a new add-on service

Step 1 — Add an entry to the addOns array in config.js:

{ id: 'add_windows', label: 'Window cleaning (interior)', price: 45 }

Step 2 — Add the new field to the booking email in Code.gs. Find the addons variable in sendBookingEmail and add a line:

d.addWindows === 'Yes' ? 'Window cleaning (+$45)' : null,

Step 3 — Add the field to the submitted data in home-cleaning.html. Find the data = { block in submitForm() and add:

addWindows: gc('add_windows') ? 'Yes' : 'No',

Redeploy Code.gs (new version), then upload config.js and home-cleaning.html to GitHub.

Change a percentage adjustment

Open config.js. All adjustments are in the middle section:

wholeHomeAdj: 0.10, // Whole home surcharge — 0.10 = 10% occupancyAdj: [ { min: 7, max: Infinity, rate: 0.30 }, // 7+ residents: +30% { min: 5, max: 6, rate: 0.20 }, // 5–6 residents: +20% { min: 3, max: 4, rate: 0.10 } // 3–4 residents: +10% ], conditionAdj: { 'Light': -0.10, // -10% 'Average': 0.00, 'Heavy': 0.20 // +20% }, deepCleanMultiplier: 1.5, // 1.5 = 50% more for deep cleans recurringDiscounts: { 'Weekly Service': 0.15, // 15% off 'Biweekly Service': 0.10, // 10% off 'Monthly Service': 0.05 // 5% off }

Change any value, save, and upload to GitHub.

Change room point values

Open config.js. Find roomPoints:

roomPoints: { bedroom: 1.00, full_bathroom: 2.00, half_bathroom: 1.00, kitchen: 3.00, living_room: 2.00, dining_room: 1.00, basement: 1.50, other_room: 1.00, closet: 0.25 }

Change any value. Save and upload to GitHub.

Add a new room type

Step 1 — Add it to roomPoints in config.js with a unique key and point value:

sunroom: 1.50

Step 2 — Add the input row to the room grid in home-cleaning.html. Find the room-grid div and add:

<div class="room-row"><div class="rl">Sunroom</div><input type="number" id="sunroom" min="0" value="0"></div>

The id must exactly match the key you added in config.js. Save and upload both files.

Add a new question to the booking form

Open home-cleaning.html. Find the Phase 2 section (it starts after the "Ready to book" checkbox). Add a new field card — copy the Additional Notes card as a template and change the label, id, and placeholder. Then add the field to the data object in submitForm():

myNewField: gv('my_field_id'),

And add it to the email body in Code.gs so it shows up in the booking alert. Redeploy Code.gs, then upload home-cleaning.html.

Update contact info (phone or email)

Open config.js and update the business block:

business: { phone: '929-274-2102', phoneHref: '9292742102', // same number, digits only email: 'drame.home.yard@gmail.com', ... }

Also update YOUR_EMAIL in Code.gs if the email changes, then redeploy Code.gs.

Update the About text

Open index.html. Find the about section and edit the paragraph text directly. Save and upload.

Change site colors

Open style.css. The entire palette is in the :root block at the very top — every color used on every page is defined there. Change any value and the whole site updates.

Apps Script — Setup & Redeployment

Where it lives

The Apps Script is attached to any Google account you choose — it does not need to be attached to a spreadsheet. You can access it directly at script.google.com.

Initial setup

  1. Go to script.google.com → New Project
  2. Name it (e.g. "Drame Brothers Email Handler")
  3. Delete the default code and paste in your Code.gs
  4. Fill in YOUR_EMAIL, BUSINESS_NAME, and BUSINESS_PHONE at the top
  5. Save (Ctrl+S / Cmd+S)
  6. Click Deploy → New deployment
    • Type: Web app
    • Execute as: Me
    • Who has access: Anyone
  7. Click Deploy — copy the Web App URL
  8. Paste the URL into config.js as the value of scriptUrl

Redeploying after changes

Any time you change Code.gs, you must create a new deployment version or the live script continues running the old code.

  1. Make your changes and save
  2. Deploy → Manage deployments
  3. Click the pencil (edit) icon on your active deployment
  4. Change Version to New version
  5. Click Deploy
The Web App URL stays the same between deployments — you do not need to update config.js when redeploying.

Testing

Two test functions are built into Code.gs. To run them: open the Apps Script editor → select the function from the dropdown → click Run.

  • testBookingEmail() — sends a sample booking alert to YOUR_EMAIL
  • testFeedbackEmail() — sends a sample feedback email to YOUR_EMAIL

If a test function doesn't appear in the dropdown, it may be nested inside another function. Scroll to the bottom of Code.gs and make sure each function starts after the closing } of the one above it.

Updating the Website (GitHub)

How to upload changes

  1. Go to your GitHub repository
  2. Click the file you want to update
  3. Click the pencil (edit) icon in the top right
  4. Make your changes — or click the upload icon to replace the file
  5. Scroll down and click Commit changes
  6. The live site updates within about 30 seconds
For most pricing changes, you only need to upload config.js. For layout or copy changes, upload the specific HTML file. You rarely need to upload multiple files at once.

Uploading multiple files

If you have several files to update at once: in the repository, click Add file → Upload files, drag all the files in at once, and commit.

Checking that changes went live

After committing, wait 30 seconds, then hard-refresh the page in your browser (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows) to bypass the cache and see the latest version.

How the Pricing Formula Works

The estimate is calculated entirely in the client's browser using the values in config.js. Here's the sequence:

  1. Room points — each room type has a point value. Multiply quantity by points for each room, sum them all up.
  2. Base price — total points × base rate ($16)
  3. Combined adjustment — add up the scope adjustment (whole home: +10%), occupancy adjustment (based on residents), and condition adjustment (Light: −10%, Average: 0%, Heavy: +20%). Apply as a single multiplier to the base price.
  4. Deep clean multiplier — for First-Time and One-Time Deep Clean, multiply by 1.5×
  5. Recurring discount — for Weekly/Biweekly/Monthly, subtract the discount percentage
  6. Flat fees — add pet surcharge ($20) if pets, then add any selected add-ons
The formula in calcPrice() in home-cleaning.html reads all values from config.js. You should never need to touch the formula itself — only the values in config.js.
